Description

Gnars Revamp: Docs and Governance

TL;DR

0.25 ETH + $900 USDC to revamp Gnars infrastructure regarding to Docs and Governance Analytics. The goal is to deliver data-driven information for DAO governance in a clear and objective way.

Introduction

Gnars DAO is going through positive changes regarding UI/UX, with a new page and features. However, some important information has become fragmented throughout the DAO's development, such as gnars.center, deprecated Documentation, deprecated Discord channels, Photos/Videos/Files built throughout the DAO's phases, Snapshot with information on the first proposals, Thats.gnarly, and Onboarding posts on Hive. This creates friction in understanding Gnars culture and difficulty with onboarding.

The goal of this proposal is to recover the DAO's history, which is fragmented across different distribution channels, smart contracts, and protocols no longer in use, in order to consolidate the DAO's storytelling in a clear and objective way, from its creation and development (Nouns Funding, Base Jumping, Zora Coins), to generate metrics that support the funding of impact ecosystems, the creator economy, retrofundings, and the funding from protocols. It makes Gnars DAO more data-driven, generating metrics to attract investment.

Scope of Work/Goals

1. Data Infrastructure & Governance Revamp

Implement a Data-Driven Governance Culture. The goal is to build a dynamic data layer that bridges Gnars' history with the novel integrations of the Nouns.build ecosystem:

  • Data Indexing: Working with the dev team (r4topunk.eth) to index governance data, DAO participation, and financial metrics (eg. revenues/expenditures), transforming them into a structured source for developers.
  • Dune Analytics: Structure the data to be more data-driven via subgraphs, facilitating integrations with the Nouns.build protocol.
  • Docs Consolidation: Unify data from Hive, Snapshot, Thats.gnarly, and Basejump into a repository for LLM and agents purposes.
  • Categorization: Implement labels like Public Goods, Athletes, DAO Ops to attach filters in future UIs.
  • Mapping Gnars Culture: Merge specs and legacy contracts to ensure that transitions to new interfaces maintain full historical continuity.
