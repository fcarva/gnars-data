Description

TL;DR

Fund the Operations team for 3 months to maintain Gnars infrastructure, enable other workstreams, and transition the DAO into an AI-native, product-oriented organization.

Why This Matters Now

Gnars is evolving from a cultural experiment into a sustainable infrastructure business. Physical products need digital storefronts. Marketing campaigns need websites that work. AI agents need organized data.

Operations is the team that builds the rails. While Marketing creates brand guidelines, we apply them to gnars.com. While Products ships physical goods, we integrate the e-commerce platform. While governance happens on Discord, we maintain community infrastructure that keeps it functional.

Without Operations, nothing else ships.

What We've Been Doing

For the past year, this team has been maintaining:

  • gnars.com (primary web presence)
  • Gnarly Agents (Bob Gnarley, Steve, community automation)
  • Discord moderation (anti-spam, anti-phishing, community support)
  • Data pipelines (Snapshot, Hive, governance archives)

We've done this work unpaid or sporadically funded because it had to be done. This proposal formalizes that work into a structured 3-month sprint.

The Shift: From Maintenance to Product

Gnars is entering a new phase:

  1. Product-centric DAO - We need infrastructure that supports physical + digital product sales
  2. AI-native operations - We're training agents to handle governance, documentation, and community support
  3. Investor-ready positioning - Clean data, clear reporting, professional tooling

Operations makes this transition possible. We're not just keeping the lights on - we're building the foundation for Gnars' next era.

3-Month Timeline

Month 1: Infrastructure Audit + Foundation

Focus: Baseline metrics, data pipeline setup, governance history consolidation

What we're building:

  • Revenue/expense tracking system - Consolidate treasury data for tokenomics analysis
  • Snapshot V1 → Base migration - Gnars governance history is fragmented (Snapshot → ETH mainnet → Base). We're consolidating everything into a unified narrative for branding and investor storytelling.
  • AI knowledge base setup - Memory system + social gateways for team usage (Bob Gnarley, Steve)
  • Data pipeline architecture - Farcaster, Hive, Discord, Snapshot → unified DB (hourly sync for active platforms, historical batch for archives)
  • Documentation mining - Extract and organize historical DAO context (meetings, decisions, deprecated areas) for AI training

Deliverable: Baseline metrics report (internal) - revenue streams, expenses, tokenomics dashboard

Month 2: Historical Archive + BI Dashboard

Focus: Public-facing tools, searchable governance data, business intelligence

What we're building:

  • Governance archive (public) - Searchable proposal history, voting records, treasury decisions (Snapshot + ETH + Base consolidated)
  • BI Dashboard - Business intelligence for DAO metrics (treasury health, proposal success rates, community activity, member retention)
  • Bob Gnarley expansion - Discord integration, governance Q&A, community support automation (trained on extracted historical context)
  • Social data integration - Farcaster feed sync (hourly), Hive content archive (historical)
  • GitHub + community tools - Organize repositories, meeting notes, deprecated code cleanup

Deliverable: Public dashboard + searchable archive - BI-grade insights for members and investors

Month 3: Product Integration + QA

Focus: E-commerce, governance analytics, testing + documentation

What we're building:

  • /shop route on gnars.com - E-commerce platform integration (Shopify or crypto-enabled solution for physical products). Platform selection based on Products workstream requirements (crypto acceptance, inventory management, international shipping).
  • Interactive governance reports - Dashboards embedded in gnars.com (proposal analytics, treasury health, voting patterns, member participation trends)
  • QA + Documentation - Test all Month 1-3 deliverables, write user docs, gather community feedback, iterate based on real usage
  • Branding application (conditional) - If Marketing delivers major design updates during Q1, we'll implement them. Otherwise, deferred to Q2.

Deliverable: Live /shop route + interactive governance dashboards + complete documentation

Why AI Matters (Context)

AI has changed how we work. Bob Gnarley (our community agent) already supports governance questions, builder assistance, and Discord moderation - but agents are only as good as their data.

This sprint focuses on making Gnars AI-digestible:

  • Data consolidation - Fragmented history (Snapshot, ETH, Base, Farcaster, Hive) → unified knowledge base
  • Memory systems - LLM-optimized documentation for instant Q&A
  • Social gateways - Team can use agents to accelerate work without hunting for context
  • Historical context mining - Extract 4 years of DAO decisions, meetings, discussions for AI training

Result: Any member (or investor) can ask an agent, "What has Gnars funded in the past 2 years?" and get a real answer.

(Example use cases: Bob Gnarley for community support, experimental governance tools like Gnars Zero Weight - but the infrastructure is what matters.)
