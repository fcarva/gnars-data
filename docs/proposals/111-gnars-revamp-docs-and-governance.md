## Summary

Proposal 111 framed the data revamp as a governance and documentation problem. The central claim was that Gnars history had become fragmented across:

- `gnars.center`
- legacy docs
- old Discord channels
- Snapshot V1
- `thatsgnar.ly`
- Hive onboarding posts
- `gnars.wtf` / Base Jump context

## Requested outputs

- Data indexing for governance participation and financial metrics
- Dune / analytics structure for a more data-driven DAO
- Consolidated docs for AI and LLM use
- Governance taxonomy for future UI filters
- Cultural continuity across legacy and current contracts

## Why it matters in this repo

Even though the proposal was defeated, it is the cleanest articulation of why `gnars-data` should exist. It is treated here as the origin brief for the repository.
