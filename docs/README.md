# Documentation

This directory holds the human-readable context around the datasets:

- `architecture/` explains how the repository is organized
- `governance/` defines proposal taxonomy and governance modeling
- `operations/` defines sync, stewardship practices, and external tooling notes
- `proposals/` archives high-signal proposal summaries

The canonical machine archive for the full proposal corpus lives in `data/proposals_archive.json`.
The proposal-tagging review queue lives in `data/proposal_tags.json`.
The canonical machine registry for verified onchain contract addresses lives in `data/contracts.json`.
The current Herd integration notes live in `docs/operations/herd-mcp.md`.
The current ETHSkills note lives in `docs/operations/ethskills.md`.
GitHub Pages publishing notes live in `docs/operations/github-pages.md`.
